## Введение

### Актуальность
Рынок игровой техники растёт: пользователи выбирают готовые сборки или собирают ПК в конфигураторе. Без единого веб-ресурса покупатель теряется в разрозненных объявлениях и чатах, не видит статус заказа и историю покупок. Нужен сайт с каталогом, корзиной, личным кабинетом и базой данных MySQL.

### Объект и предмет
- **Объект:** процессы продажи и сопровождения клиентов интернет-магазина GameTech.
- **Предмет:** веб-сайт GameTech на HTML/CSS/JavaScript + PHP + MySQL.

### Цель
Повысить удобство выбора и покупки игровой техники за счёт разработки веб-сайта с каталогом, конфигуратором, корзиной, регистрацией и хранением заказов в СУБД MySQL.

### Задачи
1. Анализ предметной области и аналогов (CMS, конструкторы).
2. Проектирование структуры сайта и БД.
3. Разработка клиентской части (витрина, ЛК, сравнение CPU).
4. Разработка серверной части (API, регистрация, заказы, заявки в поддержку).
5. Тестирование и отладка.

### Новизна и практическая значимость
Единая система: каталог комплектующих в MySQL, **сравнение процессоров** (2–3 позиции), оформление заказа с записью в `orders` / `order_items`, личный кабинет со сводкой трат, модуль заявок в поддержку (аналог «заявки плотнику» из диплома общежития).

## Технологии (как в дипломе)
- **Frontend:** HTML5, CSS3 (переменные, Grid/Flexbox), JavaScript (ES6).
- **Backend:** PHP 7.4+, mysqli, JSON API.
- **СУБД:** MySQL / MariaDB, phpMyAdmin, Open Server.
- **Безопасность:** хеш пароля (SHA-256 + соль в `auth.js`, вход в браузере), prepared statements в PHP, валидация email.

**Важно (честно):** вход на сайт и в админку — через **localStorage** (`js/auth.js`), не по паролю из phpMyAdmin. В таблице `users` поля `password_hash` и `salt` в демо-данных **пустые**. Админ `admin@gametech.ru` в MySQL **не создаётся** — только при первом открытии `admin-login.html`. Таблица `support_requests` появится после импорта обновлённого `phpmyadmin_import.sql` или файла `database/support_requests.sql`.

## Структура сайта (карта)

**Публичный уровень**
- `index.html` — главная
- `pcs.html`, `components.html`, `periphery.html` — каталог
- `configurator.html` — конфигуратор сборки
- `compare-cpus.html` — сравнение процессоров
- `contacts.html`, `delivery.html`
- `login.html` — вход / регистрация

**Для авторизованных**
- `account.html` — профиль, корзина, история закупок (MySQL + localStorage)
- `cart.html` — оформление заказа → `api/place_order.php`
- `support.html` — заявки в поддержку

**Администрирование**
- `admin.html`, `admin-login.html`
- API: `users_with_orders.php`, `support_requests.php?all=1`

## База данных `gametech`

| Таблица | Назначение |
|---------|------------|
| `components` | Каталог CPU, GPU, RAM и др. (JSON в поле `data`) |
| `users` | Учётные записи (email, имя, хеш пароля) |
| `orders` | Заказы пользователя, сумма, дата |
| `order_items` | Позиции заказа |
| `support_requests` | Заявки в поддержку (статусы: new, in_progress, done) |

Установка в phpMyAdmin: `database/phpmyadmin_import.sql` + `database/support_requests.sql`  
или: `http://gametach/api/install_users_orders.php`

## Ключевые API

| Файл | Метод | Описание |
|------|-------|----------|
| `api/components.php` | GET | Каталог по категории / по id |
| `api/register_user.php` | POST | Регистрация в БД |
| `api/place_order.php` | POST | Оформление заказа |
| `api/user_profile.php` | GET | Профиль + заказы + статистика трат |
| `api/support_requests.php` | GET/POST | Заявки поддержки |

## Сценарии тестирования (для пояснительной записки)

1. Регистрация → пользователь в таблице `users` (phpMyAdmin).
2. Добавление ПК в корзину → оформление → записи в `orders` / `order_items`, отображение в `account.html`.
3. Комплектующие → выбор 2 CPU → «Сравнить» → таблица на `compare-cpus.html`.
4. Поддержка → заявка → строка в `support_requests` → смена статуса в админке.
